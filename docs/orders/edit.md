# Карточка заказа
## Основная информация
* **Типа заказа**
    + Обычный заказ - заказ с пользователя, который требует послеющего исполнения, оплаты и доставки.
    + Коммерческое предложение - намерение пользователя в последующем оформить обычный заказ с указанными характеристиками.
* **Статус заказа**
    + Производственное состояние заказа.
* **Статус оплаты**
    + Состояние оплаты заказа.
* **Источник заказа**
    + Cайт - заказ оформлен на сайте.
    + Приложение - заказ оформлен в мобильном приложении.
    + Панель управления - заказ оформлен в панели управления.
    + Запрос по API - заказ оформлен программно.
* **Связи с заказами**
    + Ссылки на связанные заказы (например, это полезно для отслеживания заказов, требующих единой отправки; также заказы автоматически связываются при разделении сборного заказа на несколько составных).
* **Партнер**
    + Ссылка на профиль участника партнерской программы, по ссылке которого был совершен заказ.
* **Менеджер заказа**
    + Ссылка на профиль сотрудника, назначенного менеджером заказа.

## Дополнительная информация
* **Дополнительный номер**
    + По умолчанию дополнительным номер заказа совпадает с основным. Но, его можно переопределить на абсолютно любой. Иногда в дополнителном номере заказа хранят номер из финансовой или ERP-системы для сопоставления заказов (например, в случае интеграции через API).
* **Дата оформления**
    + Дата и время оформления заказа пользователем в том часовом поясе, который указан в региональных настройках сайта.
* **Менеджер выдачи**
    + Дата и время выдачи заказа пользователю, которая выставляется либо автоматически при назначении заказу статуса "Выдан", либо вручную сотрудником компании.
* **Клиент**
    + Ссылка на профиль клиента, который считается владельцем заказа. Владельца заказа можно изменить, выбрав либо произвольное частное лицо, либо зарегистрировав новое. В частности функция изменения нужна при смене типа заказа с "коммерческого предложения" на "обычный заказ".

## Примененные скидки
* **Название скидки**
    + Наименование скидки, которая были активирована для заказа.
* **Значение скидки**
    + Сумма, на которую уменьшилась стоимость заказа.

## Доступные действия
* **Смотреть обложку**
    + Обложка заказа представляет собой сборную иллюстрацию, содержащую все дизайны (или развороты) позиций заказа.
* **Скачать заказ**
    + Скачивание архива заказа с подготовленными r печати файлами. При определенных настройках архив заказа может также содержать оригиналные файлы клиента.
* **Сменить статус**
    + Смена текущего производственного статуса заказа либо на один из следующих по цепочке, либо на произвольный через пункт "Все статусы".
* **Распечатать документ**
    + Формирование сопроводительного документа по заказу на основании одного из доступных шаблонов экспорта.

## Доставка заказа
* **Получатель заказа**
    + ФИО и телефон человека, указанного в качестве получателя заказа на странице доставки. По умолчанию получатель заказа совпадает с оформителем.
* **Способ доставки**
    + Точка выдачи - собственный или партнерский офис.
    + Транспортная компания - внешняя логистическая компания.
    + Курьерская служба - собственная логистика служба.
* **Адрес доставки**
    + Полный адрес доставки или точки выдачи заказа.
* **Название доставки**
    + Название, время работы, телефон, адрес и дополнительная информация выбранного способа доставки.
* **Изменение доставки**
    + При нажатии ссылки "изменить" около заголовка открывается страница редактирования доставки.

## Редактирование доставки
* **Способ доставки**
    + Выбор способа доставки из всех доступных вне зависимости от их типа.
* **Стоимость доставки**
    + Задание стоимости доставки заказа.
* **Получатель заказа**
    + Задание имени, фамилии и телефона получателя заказа.
* **Адрес доставки**
    + Выбор собственного/партнерского офиса, точки выдачи транспортной компании или задание адреса доставку курьером.
* **Комментарий к доставке**
    + Изменение дополнительной информации по доставке (например, кода домофона или предпочтительного времени доставки).

## Оплата заказа
* **Статус оплаты**
    + Не оплачен - требуется полная оплата.
    + Частично оплачен - произведена частичная оплата.
    + Оплачен полностью - заказ оплачен полность.
* **Способ оплаты**
    + Наличные при получении.
    + Банковский перевод.
    + Счет частного лица.
    + Cчет организации.
* **Сумма оплаты**
    + Сумма внесенной предоплаты и остаток к оплате.
* **Изменение оплаты**
    + При нажатии ссылки "изменить" около заголовка открывается страница редактирования оплаты.

## Редактирование оплаты
* **Информация об оплате**
    + Статус оплаты - текущий статус оплаты.
    + Сумма заказа - итоговая стоимость заказа.
    + Личный счет - сумма на личном счете клиента.
    + Уже оплачено - сумма внесенной предоплаты.
    + К оплате - сумма, оставшаяся к оплате.
* **Внесение оплаты**
    + Сумма к оплате - вносимая сумма оплаты по последнему неоплаченному счету (в случае его отсутствия создается новый на всю сумму).
    + Способ оплаты - способ оплаты по текущему поступлению средств.
    + Коммментарий - комментарий к оплате.
* **Ожидание оплаты**
    + Сумма к оплате - сумма формируемого счета, ожидающего оплаты от клиента.
    + Способ оплаты - ожидаемый способ оплаты (данный способ будет автоматически выбран при попытке оплаты заказа клиентом через личный кабинет).
    + Плательщик - реквизиты организации, которая указана плательщиком в счете (способ оплаты - банковский перевод).
    + Получатель - реквизиты организации, которая указана получателем платежа в счете (тип организации - наша компания).
    + Коммментарий - комментарий к оплате.
* **Счета на оплату**
    + Список всех счетов (оплаченных и неоплаченных) по заказу. Любой счет можно удалить, добавить комментарий или скачать.

## Элементы заказа
* **Список позиций**
    + Позиции заказа - список товаров, содержащихся в заказа (наименование, цена за единицу, количество и сумма).
    + Стоимостные характеристики - стоимость всех позиций, скидки, бонусы, доставки, налоги, комиссии и итоговая сумма к оплате.
* **Добавить позицию**
    + При нажатии ссылки "добавить" около заголовка открывается модальное окно с добавлением новой позиции к заказу, которой может быть:
        - Продукт раздела "Печать".
        - Продукт раздела "Магазин".
        - Произвольный товар.
* **Редактировать позицию**
    + При нажатии ссылки "редактировать" около заголовка появляется возможность изменение наименования, стоимости, количества и суммы позиций.
* **Редактировать товар**
    + Запустить редактор - внести изменение в товар через редактор дизайнов или редактор проверки макетов (после внесения изменений архив заказа будет пересобран в течении 15-20 мин.).
    + Добавить комментарий - добавить комментарий, который в последствии может быть выведен в сопроводительном документе.
    + Добавить опцию - добавить к товару доступную опцию.
    + Удалить опцию - удалить опцию у товара.
    

## История заказа
* **Фильтр**
    + Фильтрация зафиксированных событий по типу:
        - Общее - изменение содержимого, оплаты или статуса заказа.
        - Доставка - изменение доставки заказа.
        - Подряд - передача заказа на подряд.
* **Лента событий**
    + Список зафиксированных событий со следующими данными:
        - Дата - дата события.
        - Статус - производственный статус заказа.
        - Имя - сотрудник, инициировавщий изменение.
        - Комментарий - системный комментарий.

## Обсуждение заказа
* **Доступные действия**
    + Смотреть макет - утвержденный клиентом макет заказа (по умолчанию совпадает с обложкой).
    + Сделить за комментариями - подписаться на получение уведомлений о появлении новых комментариев к заказу
    + Разместить комментарий - добавить новый коментарий к заказу, который может быть как публичным, так и скрытым от клиента.
* **Лента обсуждения**
    + Список сообщений клиентов и сотрудников компании, упорядоченных по времени.
    + Сотрудник компании может запросить утверждение клиентом своего сообщения (например, если он содержит предложенный вариант дизайна макета).
    + Клиент может либо утвердить сообщение сотрудника, либо отклонить его.
* **Прикрепленные файлы**
    + Список прикрепленных к заказу файлов, которые в том числе достуаны клиенту для скачивания.
    + Сотрудник компании может отметить загружаемый файл как утвержденный клиентом макет заказа.
